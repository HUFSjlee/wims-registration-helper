프로젝트 개요

프로젝트명: wims-registration-helper

목표: WIMS 양수/양도 절차를 간소화하는 웹 서비스의 프론트 UI/UX 구현

MVP 원칙

- 사용자가 직접 WIMS에 로그인/제출한다.
- 서비스는 WIMS 계정/2FA를 저장하거나 대행하지 않는다.
- 핵심 UX는 “링크 기반 정보 수집 → 한 번에 복사 → WIMS 열기 → 체크리스트로 제출 완료”.

핵심 사용자 흐름(프론트)

- 로그인 페이지 (비회원이면 회원가입 안내)
- 회원가입 페이지
- 로그인 성공 후 메인 페이지
- 메인 페이지 상단에 탭(List Tab 형태)
  - 등록(보유/증식)
  - 폐사
  - 양도/양수(조회/접수)
  - 프로필
- 양도/양수 플로우
  - (양도자) 양도 등록 화면 → 거래 생성
  - (시스템) 상대방에게 링크 공유(상대 정보 노출 최소화)
  - (양수자) 링크로 접속 → 양수 정보 입력 → 최종 컨펌
  - (제출 도우미) “WIMS 열기” + “한 번에 복사” + 체크리스트

화면 목록(프론트에서 필요한 페이지)

- /login 로그인
- /signup 회원가입
- / 메인(상단 탭)
- /transfer/new 양도 등록 화면
- /r/:token 양수자 링크 접수 화면
- /helper/:token WIMS 제출 도우미 화면 (복사 버튼 + 체크리스트 + WIMS 열기)
- /profile 로그인한 회원 프로필

UI/UX 요구사항

- 모바일 웹 우선(폰에서 쓰기 편한 UI)
- “한 번에 복사” 버튼은 사용자가 WIMS 폼에 붙여넣기 쉬운 형태로 텍스트를 생성
- 체크리스트는 제출 단계별로 체크 가능(예: 로그인 완료 / 붙여넣기 완료 / 제출 완료)
- 링크 페이지(/r/:token)는 상대방에게 전달되는 화면이므로 불필요한 민감정보 노출 금지
- 현재 단계에서는 백엔드 없이도 동작하도록 더미 데이터/로컬 상태로 구성 가능

구현 범위(프론트 MVP)

- 라우팅 + 화면 구성 + UX 동작(복사/체크/이동)
- 서버 API 연결은 추후(지금은 mock or local state)
