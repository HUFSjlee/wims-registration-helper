## 기술 스택

- **Backend**: Spring Boot
- **Database**: PostgreSQL *(현재 AWS 서버에 구축/운영 중)*
- **Architecture**: Clean Architecture 적용 예정

## 기능 목록

- **유저 회원가입** *(네이버/카카오 소셜 로그인은 일단 제외)*
    - 개인 회원
    - 사업자 회원
- **유저 로그인**
- **개체 등록(보관)**
- **양도 접수 / 양수 접수**
    - **양도자 흐름**
        - 양도자가 양도 신청
        - 마지막 단계에서 **양수자 전화번호 입력** → **링크 전송**
    - **양수자 흐름**
        - 양수자가 링크 클릭 → **양수자 정보 입력**
        - 양수자가 링크 받은 시점에 **회원가입이 안 되어있을 때**
            1. 회원가입 유도
            2. ~~비회원 처리(인증 절차 때문에 지양)~~
        - **회원가입이 되어있을 때**
            - 링크 접속 후 로그인
            - 본인 정보가 자동으로 채워짐 → 확인 후 **신청 버튼 클릭**
- **폐사 처리**
    - 숫자(수량)만 차감
- **로그인 회원 프로필 페이지**

---

## 양도 접수 화면

- 양도자의 **보유 개체 조회**
    - 여기서 바로 버튼을 눌러 **학명 표시**
    - **수량 입력**
- **일반명으로 학명 조회**
    - 예: `크레스티드게코` 입력 → 학명 노출
    - 확장 필요 *(다른 종 / 다양한 학명 매핑)*
- 마지막 단계에서 **양수자 전화번호 입력**
    - **전송하기 버튼**
        - 카카오톡 또는 문자(SMS)

---

## 양수자 접수 화면

- 양도인의 정보 중 **주소는 가린 채로 표시**
- 양수자가 **로그인 상태면 정보 자동 입력(자동 채움)**

---

## 양수자 → 양도자 처리(역전 케이스)

- 개체 수량 처리
    - **양수자 수량 증가**
    - **양도자 수량 감소**
